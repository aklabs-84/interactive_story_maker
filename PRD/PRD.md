# 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트 명**: 인터랙티브 스토리 메이커 (Interactive Story Maker)
- **목표**: 사용자가 프로그래밍 지식 없이도 선택지에 따라 분기되는 인터랙티브 스토리를 설계, 체험, 공유할 수 있는 플랫폼 제공.
- **주요 대상**: 교육자, 학생, 스토리 작가, 게임 개발 입문자.

## 2. 사용자 가용 기능 (User Features)

### 2.1. 스토리 에디터 (Editor)
- **기본 정보 입력**: 제목, 작성자, 스토리 설명을 설정할 수 있음.
- **계층형 스토리 설계**: 시작 문장으로부터 A/B 선택지를 지속적으로 확장하여 트리 구조의 시나리오를 작성.
- **이미지 업로드**: 각 장면(노드)에 시각적 이해를 돕는 이미지 추가 가능.
- **상태 관리**: 작성 중인 내용을 실시간 프리뷰로 확인하고, JSON으로 내보내거나 불러오기 가능.

### 2.2. 스토리 플레이어 (Player)
- **인터랙티브 체험**: 에디터에서 작성된 시나리오에 따라 텍스트와 이미지를 보며 직접 선택지를 고름.
- **몰입 요소**: 테마별 배경 애니메이션, 사운드 효과(BGM/SFX) 제공.
- **공유된 스토리 로드**: URL 파라미터나 JSON 데이터를 통해 타인이 만든 스토리 실행 가능.

### 2.3. 스토리 관리 (Manager)
- **로컬 저장소**: 브라우저의 LocalStorage를 사용하여 작성한 스토리 목록 자동 저장.
- **클라우드 동기화**: Google Sheets와 연동하여 데이터를 백업하고 여러 기기에서 접근 가능하게 함.
- **공유 링크 생성**: 특정 스토리를 다른 사람에게 전달할 수 있는 전용 URL 생성.

### 2.4. 테마 및 환경 설정
- **비주얼 테마**: 크리스마스, 우주 등 분위기에 맞는 테마 전환 기능.
- **다크/라이트 모드**: 사용자의 눈 피로도에 따른 모드 전환.
- **사운드 컨트롤**: 사운드 On/Off 설정.

## 3. 비기능적 요구사항
- **반응형 디자인**: 모바일, 태블릿, 데스크톱 등 다양한 기기에서 최적화된 UI 제공.
- **사용성**: 직관적인 아이콘과 토스트 메시지를 통한 피드백 제공.
- **성능**: 가벼운 바닐라 JS 기반으로 빠른 로딩 속도 유지.
